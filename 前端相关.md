## https通过AES和RSA协同工作

1. 服务器发送rsa公钥给浏览器
2. 浏览器随机生成一个aes秘钥，用服务器发送的rsa公钥加密（会话秘钥），并发送给服务器
3. 服务器通过rsa私钥解密，得到与浏览器相同的aes秘钥

## RSA签名

1. 私钥持有者对原始消息通过哈希函数生成消息摘要
2. 利用私钥加密消息摘要生成签名
3. 发送签名和原始消息
4. 接收者通过公钥解密签名，得到消息摘要1（解密失败，说明签名无效）
5. 接收者对原始消息通过哈希函数生成消息摘要2
6. 对比摘要1和摘要2（不相等说明内容被篡改）

## 获取js数据类型

```js
//简略版
Object.prototype.toString.call([]).slice(8,-1).toLowerCase()

function getType(data) {
  // 获取到 "[object Type]"，其中 Type 是 Null、Undefined、Array、Function、Error、Boolean、Number、String、Date、RegExp 等。
  const originType = Object.prototype.toString.call(data)
  // 可以直接截取第8位和倒数第一位，这样就获得了 Null、Undefined、Array、Function、Error、Boolean、Number、String、Date、RegExp 等
  const type = originType.slice(8, -1)
  // 再转小写，得到 null、undefined、array、function 等
  return type.toLowerCase()
}
```



## 单点登录、统一认证、集成登录

1. 单点登录（==SSO==）：只需在认证中心登录一次，就可以访问多个不同的系统，是一种前端用户体验
2. 统一认证：需要搭建一个==中央认证中心（IDP）==，是一种后台架构
3. 集成登录：是一种更为广泛的概念，有多种实现方式（单点登录，密码同步）。实现集成登录这个目标最推荐的方案就是 统一认证+单点登录
4. 为了达到“集成登录”的终极目标，我们通过建设“统一认证”这套基础设施，来向用户提供“单点登录”的无缝体验

## service worker

* 是一个**独立于浏览器主线程**的**脚本**，充当浏览器和网络（或缓存）之间的**代理服务器**。
* 使用场景:
  1. **离线访问**：让 Web 应用在断网时依然显示内容。
  2. **提升加载速度**：缓存常用资源，实现秒开（二次访问时直接从本地缓存加载）。
  3. **消息推送**：实现类似于原生 App 的推送通知功能。
  4. **后台同步**：在网络不稳定时，先将用户操作存储起来，等网络恢复后再发送到服务器。
